# IMPLEMENTATION_PLAN.md

## Stage 1. Анализ и архитектура
Нужно спроектировать:
- структуру Excel-книги,
- структуру листов,
- состав карточки документа,
- справочники,
- матрицы проверки,
- Word-шаблоны,
- VBA-модули,
- пользовательские формы,
- сценарии работы пользователя.

### Результат этапа
- architecture.md
- assumptions.md
- implementation_plan.md
- список рисков

## Stage 2. MVP
Реализовать:
- Excel-книгу со структурой проекта;
- базовую карточку документа;
- 2 типа документов;
- 2 Word-шаблона;
- создание документа из шаблона;
- подстановку реквизитов;
- базовую проверку обязательных полей;
- журналирование;
- экспорт DOCX/PDF.

## Stage 3. Усиление проверок
Добавить:
- проверку незаменённых маркеров;
- поиск мусорных заглушек;
- проверку обязательных разделов;
- матрицу clauses для Engineering Analysis;
- проверку attachments / references;
- форму отчёта замечаний.

## Stage 4. UX и поддерживаемость
Добавить:
- удобную UserForm;
- понятные кнопки запуска;
- сообщения об ошибках;
- комментарии в коде;
- конфигурационный лист;
- инструкции для пользователя и разработчика.

## Stage 5. Тестирование и выпуск
Подготовить:
- test plan;
- sample data;
- ручной чек-лист приёмки;
- перечень ограничений;
- рекомендации по дальнейшему развитию.
