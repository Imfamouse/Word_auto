# VALIDATION_RULES.md

## Уровни проверок

### 1. Ошибки-блокеры
При наличии блокирующих ошибок выпуск документа в PDF не должен считаться завершённым.

Блокеры:
- не заполнены обязательные поля карточки документа;
- не найден или не выбран Word-шаблон;
- в документе остались маркеры вида {{...}};
- в критичных местах документа обнаружены пустые значения;
- отсутствуют обязательные разделы;
- для applicable clauses в Engineering Analysis отсутствует status;
- для applicable clauses отсутствует means of compliance;
- для applicable clauses отсутствует ссылка на раздел документа;
- в документе обнаружены противоречивые значения ключевых реквизитов.

### 2. Предупреждения
Предупреждения не всегда блокируют выпуск, но требуют внимания инженера или проверяющего.

Предупреждения:
- найдены слова TBD / XXX / ??? / sample / draft;
- в тексте есть ссылки на figures / attachments, которые не подтверждены;
- есть sections, помеченные как условно обязательные, но не подтверждённые правилом;
- в журнале не указан checker / approver;
- подозрительно пустые разделы.

## Критичные реквизиты для сверки
- document_id
- document_type
- aircraft_number
- msn
- assembly_number
- part_number
- component_name
- revision
- date
- applicability

## Специальные проверки для Engineering Analysis
Для каждой строки матрицы clauses, где applicability_flag = true:
- status обязателен;
- means_of_compliance обязателен;
- covered_in_section обязателен;
- при необходимости evidence_reference обязателен;
- статус не должен быть undefined / blank.

## Специальные проверки для Repair Instruction
- обязательные разделы присутствуют;
- условно обязательные разделы включаются по правилу;
- references и attachments проверяются, если они упоминаются;
- инженерный текст не должен быть автоматически перезаписан при синхронизации реквизитов.

## Минимальный итог отчёта
Отчёт должен показывать:
- тип замечания,
- severity,
- место обнаружения,
- краткое описание,
- рекомендуемое действие.
