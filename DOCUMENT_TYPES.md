## 1. Repair Instruction (Ремонтная инструкция)

### Назначение
Документ описывает ремонтное решение, применяемость, материалы/инструменты, последовательность выполнения, проверки и ограничения.

### Важное ограничение
Содержательная часть ремонта может сильно различаться от случая к случаю.
Поэтому система **не** пытается автоматически писать repair procedure.

### Что автоматизируется
- реквизиты документа (номер, ревизия, дата);
- данные объекта ремонта (тип ВС, MSN, рег. номер, данные наработки);
- данные компонента (обозначение, серийный номер, наработка);
- титул, таблица подписей, колонтитулы;
- лист регистрации изменений;
- проверка на незаполненные маркеры `{{...}}`;
- проверка на незаполненные секции `<<...>>`;
- проверка на мусорные заглушки.

### Структура документа (на основе `A.095.04.00000.078.0699_example.docx`)

```
[Титульная страница]
  Компания / Сертификат
  Тип ВС. Заголовок ремонта
  Номер документа / Номер изменения
  Таблица подписей: Разработал / Прочность / Утвердил

[Содержание]

Лист регистрации изменений
  Таблица: Изм. | Дата | Страницы | Описание

1. Вводная часть
   1.1 Общие положения        ← инженер пишет вручную
   1.2 Единицы измерения      ← фиксированный текст
   1.3 Термины и сокращения   ← инженер пишет вручную

2. Ссылочная документация
   2.1 Общая документация     ← таблица, инженер заполняет

3. Информация о ремонте
   3.1 Описание повреждения   ← инженер пишет вручную
   3.2 Описание ремонта       ← инженер пишет вручную

4. Информация для планирования работ
   4.1 Исполнитель работ
   4.2 Применяемость          ← таблица: {{AircraftModel}}, {{MSN}}, {{ComponentName}} и т.д.
   4.3 Изменение массы и центровки
   4.4 Инструкции по ПЛГ      → ссылка на раздел 6

5. Указания по выполнению ремонта
   5.1 Выполнение осмотра     ← инженер пишет вручную
   5.2 Заключительные работы  ← инженер пишет вручную

6. Инструкции по поддержанию летной годности
   Таблица: категория, ресурс, интервал, начало инспекций,
            метод, содержание инспекции   ← инженер заполняет

7. Уведомление о выполнении ремонта
   Форма отправки уведомления

8. Обратная связь
   Форма замечаний и оценки
```

## 2. Engineering Analysis (Инженерный анализ)

### Назначение
Документ обосновывает соответствие конструкции, ремонта или решения применимым пунктам базиса.

### Важное ограничение
Логика анализа, расчёты, применимость и набор пунктов базиса могут существенно меняться.
Поэтому система **не** пытается автоматически писать reasoning или расчётную часть.

### Что автоматизируется
- реквизиты документа;
- общая структура;
- перечень clauses;
- матрица покрытия базиса;
- наличие статусов по applicable clauses;
- наличие means of compliance;
- наличие ссылок на разделы и подтверждающие материалы;
- проверка полноты покрытия и согласованности.

### Статус шаблона

⚠ **`EngineeringAnalysis.dotx` является текстовым файлом-заглушкой и требует замены**  
на полноценный OOXML-шаблон, аналогичный `RepairInstruction.dotx`.

### Типовые обязательные разделы
- Objective / Scope
- Description of design / repair
- Inputs / assumptions
- Certification basis / applicable clauses
- Means of compliance
- Analysis
- Conclusion
- Limitations / remarks
- References / attachments

## Общий принцип

Word-шаблон задаёт каркас документа.  
Инженер пишет техническое содержание вручную.  
Excel хранит карточку документа, правила и матрицы проверки.  
VBA автоматизирует подстановку реквизитов, валидацию и выпуск.
