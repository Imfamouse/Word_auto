# NAMING_RULES.md

## Общие принципы
- Имена файлов должны формироваться автоматически.
- Пользователь не должен вручную вводить длинные служебные имена файлов.
- Формат должен быть читаемым и одинаковым для всех типов документов.

## Шаблон имени
Правила составления имен лежат в naming_rules.xlsx

## Примеры
Ремонтная инструкция А.737.04.30102.057.0001
Инженерный анализ VDE.737.04.30102.057.110.0001.31

## Правила
- не использовать пробелы, если это не требуется внутренним стандартом;
- revision должна браться из карточки документа;
- PDF и DOCX должны иметь одинаковую базовую маску имени;
- при повторной генерации файл не должен молча затирать предыдущий выпуск без явного подтверждения или правила версии.

## Пути хранения
Пути к:
- шаблонам,
- рабочим DOCX,
- итоговым PDF,
- журналам
должны храниться в конфигурации, а не в коде.
