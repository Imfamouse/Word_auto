# План тестирования

## Позитивные сценарии

- Создание карточки RI и генерация DOCX из корректного шаблона.
- Создание карточки EA и генерация DOCX из корректного шаблона.
- Подстановка всех маркеров реквизитов.
- Успешная проверка без ошибок и экспорт PDF.

## Негативные сценарии

- Пустые обязательные поля карточки.
- Отсутствует файл шаблона.
- В документе остались `{{...}}`.
- В документе есть `TBD`/`XXX`/`???`/`sample`/`draft`.
- Отсутствуют обязательные разделы по `rules_required_sections`.
- Для EA есть applicable clause без метода закрытия.
- Для RI по условию отсутствует требуемый раздел.

## Регрессия

- Повторная генерация документа не ломает исходный шаблон.
- Журналирование работает на всех пользовательских операциях.
- Экспорт PDF не ломает путь DOCX.

## Чек-лист ручной приёмки

- [ ] Карточка создаётся и сохраняется.
- [ ] Документ создаётся по правильному шаблону.
- [ ] Реквизиты синхронизированы в титуле и колонтитулах.
- [ ] Валидация показывает понятные ошибки/предупреждения.
- [ ] PDF успешно экспортируется.
- [ ] В `log_actions` отражены ключевые действия.
