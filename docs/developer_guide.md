# Руководство разработчика

## Принципы

- Не генерировать инженерное содержание автоматически.
- Не смешивать UI с бизнес-правилами.
- Все листы, маркеры и коды — через константы.
- Новые типы документов добавляются через справочники и правила.

## Как расширить на новый тип документа

1. Добавить тип в `ref_document_types`.
2. Добавить шаблон в `ref_templates`.
3. Добавить обязательные поля в `rules_required_fields`.
4. Добавить обязательные разделы в `rules_required_sections`.
5. При необходимости добавить специализированные проверки в `modValidation`.

## Импорт VBA

- Модули: импорт из `vba/modules/*.bas`.
- Классы: импорт из `vba/classes/*.cls`.
- Формы: импорт из `vba/forms/*.frm`.

## Обработка ошибок и COM

- `modWordAutomation` использует late binding.
- Любая процедура с Word имеет блок `CleanUp` и `Set ... = Nothing`.
- Шаблон никогда не открывается на запись; создаётся новый документ на его основе.
