# Ограничения и roadmap

## Текущие ограничения

### Критические

- **`EngineeringAnalysis.dotx` — текстовый файл-заглушка.**  
  Шаблон EA не является валидным OOXML и не будет работать с `modWordAutomation`.  
  Требуется создать полноценный `.dotx` на основе реального примера EA-документа.

### Архитектурные

- Без серверной части нет централизованной блокировки одновременного редактирования одного документа двумя пользователями.
- `modMain.CreateWordDocument` / `ValidateCurrentDocument` / `ExportCurrentToPdf` читают данные из открытой формы. Если форма не открыта — получают пустые значения. Основной сценарий работы — через кнопки внутри `frmDocumentCard`.
- Загрузка строки из `doc_cards` при открытии формы берёт `ActiveCell.Row`. Если курсор не на нужной строке — загрузятся не те данные.

### Функциональные

- Проверка ссылок на attachments/references основана на ключевых словах и может требовать уточнений.
- Условно обязательные разделы RI (с `condition_rule`) реализуются в табличной форме и требуют дисциплины заполнения листа `ri_section_matrix`.
- Нет автоматического обновления таблицы содержания в Word-документе (обновляется вручную при открытии / `Ctrl+A → F9`).
- Нет diff при создании новой ревизии — лист изменений заполняется вручную.

## Roadmap

### Приоритет 1 — необходимо для полноценного MVP

- [ ] Создать `EngineeringAnalysis.dotx` по аналогии с RI: взять реальный пример EA, расставить маркеры.
- [ ] Добавить кнопки сценариев на лист `ui_dashboard`.

### Приоритет 2 — улучшение UX

- [ ] Визуальная группировка полей формы по секциям (Aircraft / Component / Document / Persons).
- [ ] Выпадающие списки для полей `document_type`, `status`, `author`, `checker`, `approver` из справочников.
- [ ] Автозаполнение даты текущей датой при создании новой карточки.
- [ ] Защита от перезаписи существующего DOCX без подтверждения.

### Приоритет 3 — расширение функциональности

- [ ] Diff ревизий: при создании новой ревизии предлагать заполнить строку в листе изменений автоматически.
- [ ] Добавить цифровую подпись и контроль неизменности после выпуска (`Released`).
- [ ] Пакетный режим проверки нескольких документов.
- [ ] Экспорт отчёта валидации в отдельный документ Word/PDF.

### Приоритет 4 — новые типы документов

- [ ] Механизм добавления нового типа документа без изменения ядра (только справочники + шаблон).
- [ ] Расширенный шаблон заголовков для документов с несколькими ВС.
